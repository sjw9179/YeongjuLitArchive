# YeongjuLitArchive

YeongjuLitArchive는 대규모 언어 모델(LLM)을 활용하여  
문학 작품(시·산문)을 분석하고,  
그 결과를 정서 흐름, 주제, 표현 기법, 주석 등으로 구조화해
인터랙티브하게 시각화하는 웹 애플리케이션입니다.

기존의 키워드 사전이나 규칙 기반 분석 방식을 사용하지 않고,
AI의 의미 기반 해석 결과를 JSON 형태로 받아
프론트엔드에서 그대로 렌더링하는 구조를 채택했습니다.

---

## ✨ 주요 기능

- 📖 **인터랙티브 문학 주석**

  - 작품 내 핵심 표현 자동 강조
  - 마우스 오버 시 문학적 의미 설명 표시

- 📊 **정서 흐름 시각화**

  - 연/문단 단위 정서 변화 분석
  - 그래프를 통한 감정선 표현

- 🧠 **AI 기반 문학 분석**

  - 주제, 정서, 화자의 태도, 작품 요약 자동 분석
  - 갈래, 시점, 운율 등 메타데이터 정리

- 🧩 **JSON 중심 설계**
  - 키워드 사전, 하드코딩 규칙 사용 ❌
  - AI 분석 결과(JSON)를 그대로 UI에 반영

---

## 🛠 사용 기술

- **프론트엔드**

  - HTML / CSS / JavaScript
  - GSAP (애니메이션)
  - Chart.js (정서 흐름 시각화)

- **AI / 분석**
  - Groq API
  - GPT 계열 대규모 언어 모델(LLM)

---

## 🔍 동작 방식

1. 사용자가 문학 작품을 입력
2. 입력 텍스트를 AI 분석 API로 전송
3. AI가 작품을 분석해 구조화된 JSON 데이터 반환
4. 프론트엔드는 해당 데이터를 기반으로:
   - 핵심 표현 강조
   - 주석 표시
   - 정서 흐름 그래프 생성
   - 작품 메타데이터 출력

## 🎯 제작 동기

기존의 문학 분석 도구는  
키워드 빈도나 단순 감정 점수화에 의존하여  
작품의 맥락, 상징, 정서를 충분히 반영하지 못하는 한계가 있었습니다.

본 프로젝트는  
AI의 의미 기반 해석을 활용해  
문학 작품을 보다 구조적으로 이해할 수 있는 방법을 탐구하고자 시작되었습니다.

## 📌 교육적 활용

이 프로젝트는 학생 탐구 및 학습 목적의 연구 프로젝트입니다.

AI의 분석 결과를 그대로 받아들이는 것이 아니라,

- 인간의 해석과 비교
- AI 해석의 한계 인식
- 질문(프롬프트) 설계의 중요성

등을 함께 탐구하는 데 목적이 있습니다.

## ⚠️ 한계 및 유의점

- 문학 해석은 본질적으로 주관적 요소를 포함함
- 본 프로젝트는 분석 도구이지 정답 제공 도구가 아님
